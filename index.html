🎯 Planteamiento del Problema

💡 Necesidad: Desarrollar un sistema de control de acceso tecnológicamente robusto pero económicamente accesible para comunidades de estratos medios.

Contexto: Conjuntos residenciales de estratos medios en Pasto, Nariño

Problemáticas Identificadas:

  • Seguridad deficiente: Sistemas manuales propensos a errores y falsificaciones
  • Ineficiencia operativa: Dependencia de personal 24/7 genera altos costos
  • Limitaciones tecnológicas: Sistemas comerciales cuestan entre $1.5M - $5M pesos
  • Falta de trazabilidad: No hay registro confiable de ingresos y salidas
2/12

🎯 Objetivos del Proyecto

🎯

Objetivo General:

⚙️ Funcionamiento - Entrada

Sistema de Control Peatonal:

🔢 Método 1: Teclado Numérico

Entrada: Usuario ingresa código de 4 dígitos (ej: 1012)

Proceso: ESP32 compara código ingresado con contraseña almacenada

Salida: Si es correcto → Servo abre puerta + LED enciende + Buzzer suena 1 vez

Seguridad: Bloqueo automático después de 5 intentos fallidos por 1 minuto

📱 Método 2: Tarjeta/Llavero RFID

Entrada: Usuario acerca tarjeta RFID al sensor (2-4 cm)

Proceso: Sensor lee UID único de tarjeta y ESP32 lo compara con tarjetas autorizadas

Salida: Si está autorizada → Misma respuesta que teclado

5/12

⚙️ Funcionamiento - Proceso

Sistema de Control Vehicular:

🚗 Control Remoto Infrarrojo

Entrada: Usuario presiona botón del control remoto

Proceso: Sensor VS1338B detecta señal IR, ESP32 decodifica comando

Salida: Servo vehicular abre/cierra barrera según botón presionado

Alcance: Hasta 4 metros en interiores

Programación:

Lenguaje: C++ en Arduino IDE 2.3.6

Arquitectura: Procesamiento doble-núcleo del ESP32-S3

  • Core 1: Maneja acceso peatonal (teclado + RFID + pantalla)
  • Core 1: Maneja acceso vehicular (control remoto)

Protocolos: I2C (pantalla OLED), SPI (sensor RFID), PWM (servos), IR (control remoto)

🔨 Proceso de Elaboración - Fase 1

1. Investigación y Planificación (Semanas 1-2):

  • Identificación del problema en conjuntos residenciales de Pasto
  • Análisis de alternativas: Arduino UNO vs Raspberry Pi vs ESP32-S3
  • Selección de componentes y elaboración de presupuesto

2. Configuración del Entorno (Semanas 3-4):

  • Instalación de Arduino IDE 2.3.6 y drivers CP210x
  • Configuración de soporte ESP32 e instalación de librerías
  • Compra de materiales en tiendas locales de Pasto

3. Desarrollo de Componentes (Semanas 5-7):

  • Prueba individual de cada sensor y actuador
  • Calibración de servomotores y sensores
  • Programación de interfaces y lógica básica
8/12

🔨 Proceso de Elaboración - Fase 2

4. Integración de Subsistemas (Semanas 8-10):

  • Integración teclado + OLED + servo peatonal
  • Integración RFID + sistema peatonal
  • Implementación procesamiento doble-núcleo
  • Resolución de conflictos de pines y recursos

5. Construcción de Maqueta (Semanas 11-12):

  • Diseño y construcción de estructura en cartón paja
  • Montaje de puertas y mecanismos con servos
